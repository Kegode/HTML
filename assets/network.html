What is a Network?

  • Two or more devices connected to share resources
  • Examples: internet, office LAN, home Wi-Fi
  • Key benefits: file sharing, communication, central management

LAN – Local Area Network

Small & Fast
  • Limited area: home, school, office building
  • High speed (1–10 Gbps)
  • Privately owned
  • Low latency
🏢 Diagram of a small office with PCs, printer, switch, and router
4 / 30

WAN – Wide Area Network

Large & Global
  • Spans cities, countries, continents
  • Internet = largest WAN
  • Slower than LAN due to distance
  • Uses leased lines or satellite links
🗺️ World map with lines connecting cities (New York → London → Tokyo)
5 / 30

MAN – Metropolitan Area Network

City or Campus
  • Covers a city or large university campus
  • Larger than LAN, smaller than WAN
  • Often owned by a single organization or ISP
🏙️ City skyline with fiber optic rings connecting buildings
6 / 30

PAN – Personal Area Network

Very Short Range
  • Around a single person (few meters)
  • Examples: Bluetooth headset, phone ↔ laptop, USB
  • Low power, low cost
👤 Person with smartphone, smartwatch, wireless earbuds, and laptop all connected wirelessly

Network Topologies Overview

  • Star: All devices to central switch
  • Bus: Single cable backbone
  • Ring: Each device to two others
  • Mesh: Fully or partially interconnected

TCP/IP vs OSI Model

  • OSI: 7 layers (theoretical, teaching)
  • TCP/IP: 4 layers (practical, internet)
  • Both describe data flow but differ in granularity
🔄 Side-by-side layer diagrams with arrows showing mapping between layers
10 / 30

OSI Upper Layers (7–4)

  • L7 Application: User apps (HTTP, FTP)
  • L6 Presentation: Encryption, compression
  • L5 Session: Manages connections
  • L4 Transport: Reliable delivery, TCP/UDP
📚 7-layer pyramid with top 4 layers highlighted
11 / 30

OSI Lower Layers (3–1)

  • L3 Network: IP addressing, routing
  • L2 Data Link: MAC addresses, switches, frames
  • L1 Physical: Cables, Wi-Fi, bits
🔌 7-layer pyramid with bottom 3 layers highlighted

Data Encapsulation

  • Data → Segment → Packet → Frame → Bits
  • Each layer adds its own header
  • Opposite process at receiving end (de-encapsulation)

Public vs Private IP Addresses

  • Public: Unique on internet, assigned by ISP
  • Private: For internal networks (192.168.x.x, 10.x.x.x, 172.16.x.x)
  • NAT translates private → public

IPv6 – The Future

  • 128-bit → 340 undecillion addresses
  • Example: 2001:0db8:85a3::8a2e:0370:7334
  • No NAT needed, built-in security
  • Hexadecimal notation

DNS – Domain Name System

The Internet's Phonebook
  • Translates names (google.com) → IPs (142.250.190.46)
  • Distributed database (no single point of failure)
  • Makes internet human-friendly
📞 Smartphone contacts list next to a server labeled "DNS"
19 / 30

How DNS Works – Step by Step

  1. User types www.google.com
  2. DNS Resolver checks local cache
  3. Queries Root Server
  4. Queries TLD Server (.com)
  5. Queries Authoritative Server
  6. Returns IP address
  7. Browser connects

TCP – Transmission Control Protocol

Reliable & Ordered
  • Connection-oriented (3-way handshake)
  • Guaranteed delivery (ACKs)
  • Data arrives in order
  • Error checking & correction
  • Flow control (window size)
  • Slower due to overhead
🤝 Two computers shaking hands (3-way handshake) with checkmark icon
24 / 30

UDP – User Datagram Protocol

Fast & Simple
  • Connectionless (no handshake)
  • No delivery guarantee
  • Order not guaranteed
  • Minimal error checking
  • No flow control
  • Faster, low overhead
🚀 Rocket ship next to a packet being sent without confirmation

Week 2 Networking Summary

  • Networks range from PAN (personal) to WAN (global)
  • TCP/IP (4 layers) and OSI (7 layers) model data flow
  • IP addresses uniquely identify devices (IPv4 and IPv6)
  • DNS translates human names to IPs
  • HTTP/HTTPS powers the web
  • TCP = reliable & slow, UDP = fast & simple
